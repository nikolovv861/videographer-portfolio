<task type="auto">
  <name>Task 1: Create CustomCursor component with dot + spring ring follower</name>
  <files>components/effects/CustomCursor.tsx</files>
  <action>
Create `components/effects/CustomCursor.tsx` -- "use client" component.

**Imports:** motion, useMotionValue, useSpring from "motion/react". useState, useEffect from "react".

**Touch device detection:**
Create a hook `useIsDesktopPointer()`:
- Uses `window.matchMedia("(hover: hover) and (pointer: fine)")` to detect fine pointer (mouse)
- Returns boolean, starts false (SSR-safe), updates on mount and on media query change
- If not desktop pointer, component renders null (hidden on touch)

**Mouse tracking:**
- `cursorX` and `cursorY` as `useMotionValue(0)` for direct dot position
- `ringX` = `useSpring(cursorX, { stiffness: 150, damping: 15 })` for delayed ring X
- `ringY` = `useSpring(cursorY, { stiffness: 150, damping: 15 })` for delayed ring Y
- Stiffness 150 / damping 15 gives ~100-150ms springy follow per user decision
- Add mousemove event listener on window, update cursorX/cursorY with `e.clientX`/`e.clientY`

**Visibility:**
- Track `isVisible` state, starts false
- Set to true on first mousemove event (prevents cursor flash at 0,0 on page load per pitfall #2)
- Both dot and ring have `opacity: isVisible ? 1 : 0`

**Hover detection:**
- Track `isHovering` state
- In the mousemove handler (or a separate mouseover/mouseout listener on document), check if `e.target` (or its ancestors) matches interactive elements: `a, button, [role="button"], input, textarea, select, label, [data-cursor-hover]`
- Use `(e.target as HTMLElement).closest('a, button, [role="button"], input, textarea, select, label, [data-cursor-hover]')` to detect hover
- When hovering: ring scales to 1.5x, fills with bg-gold/30 (semi-transparent champagne gold), dot scales to 0 (shrinks/disappears)
- When not hovering: ring at scale 1 with border-only, dot visible

**Rendering:**
```
Dot: pointer-events-none, fixed positioning, z-[90], w-2 h-2, rounded-full, bg-foreground (#f5f5f5), -translate-x-1/2 -translate-y-1/2, style={{ left: cursorX, top: cursorY }}

Ring: pointer-events-none, fixed positioning, z-[90], w-10 h-10, rounded-full, border border-foreground/60, -translate-x-1/2 -translate-y-1/2, style={{ left: ringX, top: ringY }}, animate={{ scale: isHovering ? 1.5 : 1, backgroundColor: isHovering ? "rgba(201,169,110,0.3)" : "transparent" }}
```

**Reduced motion:** Check `window.matchMedia("(prefers-reduced-motion: reduce)")`. If enabled, render null (no custom cursor). Users with reduced motion keep native cursor.

Export: `CustomCursor`

1. In `app/layout.tsx`:
   - Import CustomCursor from "@/components/effects/CustomCursor"
   - Render `<CustomCursor />` inside the body, after ActiveSectionProvider (as a sibling, not inside provider). It's a global component that renders on every page.
   - Place it right before the closing `</body>` or just inside body after the provider block.

2. In `app/globals.css`, add a rule to hide the native cursor on desktop:
   ```css
   @media (hover: hover) and (pointer: fine) {
     * {
       cursor: none !important;
     }
   }
   ```
   This hides the native cursor only on devices with fine pointer (mouse), preserving normal cursor on touch devices. Place this BEFORE the prefers-reduced-motion rule. Note: the reduced-motion rule doesn't affect cursor:none since it's not an animation -- but the CustomCursor component itself returns null for reduced-motion users, so native cursor behavior will need to be restored. Add an additional rule:
   ```css
   @media (prefers-reduced-motion: reduce) {
     * {
       cursor: auto !important;
     }
   }
   ```
   This restores the native cursor for reduced-motion users since the CustomCursor component won't render.

3. Run `npm run build` to verify no build errors.

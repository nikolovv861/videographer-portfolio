  <files>components/ui/VimeoEmbed.tsx, app/layout.tsx, app/page.tsx</files>
  <action>
1. Update `components/ui/VimeoEmbed.tsx` to support lazy loading:
   - Add "use client" directive (needed for useInView hook)
   - Import `useInView` from "react-intersection-observer"
   - Add optional `lazy?: boolean` prop to VimeoEmbedProps (default: false for backward compat)
   - When `lazy` is true:
     - Use `const { ref, inView } = useInView({ triggerOnce: true, rootMargin: "200px" })` to detect when component is near viewport
     - Wrap the existing iframe in a conditional: if `inView`, render the iframe; if not, render a placeholder div with bg-background matching the container dimensions
     - Apply ref to the outer container div
   - When `lazy` is false (default): render iframe immediately as before (no behavioral change for Hero which needs eager loading)
   - The Hero video must NOT be lazy-loaded (it's above the fold and needs to play during loading intro)

2. Update consumers to pass `lazy={true}`:
   - In `components/sections/Showreel.tsx`: add `lazy` prop to VimeoEmbed usage
   - In `components/sections/FeaturedWork.tsx` (or ProjectCard.tsx): if VimeoEmbed is used, add `lazy` prop
   - Do NOT add lazy to Hero.tsx (above fold, needs eager loading)
   - For ProjectModal.tsx: the inline iframe can remain eager since modal only opens on user action (already effectively lazy by being hidden)

3. Enhance `app/layout.tsx` metadata export:
   - Expand the existing `metadata` object with:

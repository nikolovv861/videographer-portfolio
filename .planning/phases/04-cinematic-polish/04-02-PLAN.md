  <name>Task 1: Create LoadingIntro component with full letterbox cinema sequence</name>
  <files>components/effects/LoadingIntro.tsx</files>
  <action>
Create `components/effects/LoadingIntro.tsx` -- "use client" component using motion from "motion/react" (AnimatePresence, motion, animate or useAnimate).

**Component logic:**
1. On mount, check `sessionStorage.getItem("hasVisited")`:
   - If NOT set: first visit mode (full 2.5-3s sequence). Set `sessionStorage.setItem("hasVisited", "true")`.
   - If set: return visit mode (0.5s quick open only).

2. State: `isComplete` (boolean, starts false). When true, overlay unmounts via AnimatePresence.

**First visit sequence (2.5-3s total):**
The overlay is a fixed div at z-[200], bg-[#0a0a0a], covering full viewport.

Stage 1 (0-0.4s): A single thin horizontal line (1-2px height, bg-gold #c9a96e) appears dead center of viewport. Starts at scaleX(0), animates to scaleX(1) -- center-outward expansion. Use transformOrigin: "center".

Stage 2 (0.4-1.0s): The single line morphs into two letterbox bars. Top bar animates upward to approximately top-third position, bottom bar animates downward to approximately bottom-third position. Both bars are full-width, ~80-100px tall, bg-gold initially then transition to a dark charcoal matching background. The gap between them mimics a cinematic 2.39:1 aspect ratio. During this transition, a subtle "slit" of the hero video behind becomes visible between the bars.

Stage 3 (0.8-1.5s): Videographer wordmark/logo text ("Alex Rivera" from siteConfig.name) fades in at absolute center in off-white (#f5f5f5), using font-heading (Syne). Very subtle, not large -- tasteful. Use the data import: `import { siteConfig } from "@/data/navigation"`.

Stage 4 (0.3-2.5s): Percentage counter in bottom-right corner (fixed, small text, warm gray #a0a0a0). Counts from 0 to 100 with an ease-out curve (fast through middle numbers, slows near 100). Use useEffect with requestAnimationFrame or a motion value to animate the count. Display as "XX%" format.

Stage 5 (2.5-3.0s -- at 100%): Reveal sequence:
  - Logo scales up slightly (1.0 -> 1.05) and fades out
  - Counter fades out
  - Top bar slides upward off-screen (translateY: -100%)
  - Bottom bar slides downward off-screen (translateY: 100%)
  - Set isComplete to true after exit animation

**Visual effect during stages 2-4:** The hero video visible through the slit between letterbox bars should appear desaturated and slightly blurred. Achieve this by applying a CSS filter on the overlay's "window" area -- since the overlay covers everything except the slit gap, the hero video peeks through. To create the desaturation/blur effect, add a semi-transparent layer with backdrop-filter: blur(2px) grayscale(0.8) positioned in the gap between bars. At reveal (stage 5), this filter layer animates to blur(0) grayscale(0) before bars slide away -- the "coming into focus" moment.

**Film grain:** Import and render FilmGrain component from "@/components/effects/FilmGrain" inside the overlay div (grain visible throughout intro per user decision).

**Return visit sequence (0.5s):**
- Show only the two letterbox bars already in position (no line expansion, no logo, no counter)
- Bars immediately start sliding away (top up, bottom down)
- 0.5s total duration, then isComplete = true

**Reduced motion:** Check `window.matchMedia("(prefers-reduced-motion: reduce)")`. If enabled, skip all animations -- set isComplete to true immediately (or after a minimal 100ms delay for paint).

**Important implementation notes:**
- The overlay sits ABOVE the hero video (z-200 vs hero at z-0). The hero video loads and plays behind the overlay from the start -- no blocking of video buffering.
- Use AnimatePresence to handle the exit animation when isComplete becomes true.
- Keep timing constants as named variables at the top of the file for easy tuning.
- Do NOT import or play any video in this component -- the hero video is already playing in the Hero component behind.
